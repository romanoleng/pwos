# What about bank statements?

**Not needed for the reset, and they would slow it down.** The reset is about
positions; a statement is about history.

**Where a statement genuinely helps is a different job:** checking that PWOS's
ledger matches what the bank actually saw. Right now PWOS has ~64 transactions a
month. If your real card usage is higher, the ledger has gaps — and budgets
would be understating spend without either of us knowing.

That is worth doing, and it needs **CSV import + reconciliation**, which is on
the V1.1 list and not built. If you want it, the useful thing to send is a
**Capitec CSV export** (not PDF) covering 24 Jun → 24 Jul, so I can compare it
line by line against what PWOS holds for the same period and tell you what is
missing.

**Suggested order:** reset Friday so the app is correct from payday, then decide
about statement import once you have seen a full clean cycle. Doing both at once
means you would not know which one fixed or broke a number.
